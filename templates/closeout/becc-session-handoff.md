# becc-session-handoff

用于把当前会话阶段性结果交接给后续继续推进的人或下一次会话。

## 模板

```md
## Completed
- 

## Changed
- 

## Verified
- 

## Not Run
- 

## Risks / Open Questions
- 

## Recommended Next Steps
1. 
2. 
3. 

## Resume Point
- First file / result / question to inspect next:

## Reviewer Focus（如适用）
- 
```

## 使用要求

- 已完成与未完成必须分开
- Not Run 不能省略
- Next Steps 要能直接接着做，而不是空泛建议
- Resume Point 要尽量降低下一次会话的上下文重建成本
