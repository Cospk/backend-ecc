# becc-problem-statement

用于把一个任务先收敛为最小问题定义，再决定是否进入证据、scope、design 或 debug。

## 模板

```md
## Problem Statement
- Current task:
- Target object:
- In-scope:
- Out-of-scope:

## Known Constraints
- Technical:
- Business:
- Environment / process:

## Unknowns
- 
- 

## Success Criteria
- 
- 

## Recommended Next Step
- evidence capture | impact discovery | debug root cause | direct execution
```

## 使用要求

- 不要把未知点省略掉
- 不要把成功标准写成空泛目标
- 如果下一步仍不明确，应回到补信息，而不是直接进入实现
