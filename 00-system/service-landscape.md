# 服务地图

这份文档用于从系统级视角整理服务清单、职责边界和上下游关系。它不展开代码实现，只回答系统中“谁负责什么、依赖谁、输出给谁”。

### 我们负责的业务系统

## 当前重点：推送组相关服务

### market-server
- 角色：推送组中的盘口处理与分发服务组
- 主要职责：盘口相关 API、异步事件处理、Redis PubSub 与 Kafka 分发
- 运行形态：server / consumer

### push-server-gw

- 角色：面向操盘页面推送数据
- 主要职责：接收各组数据发送的pubsub数据，通过ws协议推送操盘页面

### c-push-server
- 角色：推送汇聚层
- 主要职责：接收各组数据，整理后继续向下游 Kafka 推送

### go-stomp
- 角色：面向 95e 的推送出口
- 主要职责：消费 `c-push-server` 数据，通过  api / stomp 协议向用户推送

### c-server
- 角色：待确认
- 当前推测：偏向服务 95e，可能维护用户信息与用户配置
